## Anti-Patterns Verdict

**LLM assessment**: Concerns, not clean. The Darkroom palette and draft-state badges show clear design intent and discipline. But there are four simultaneous visual conflicts undermining it. The uppercase eyebrow pattern (23 instances) is the dominant AI-template tell: every panel, every section, every status group has a small 11px all-caps label as scaffolding. The ghost-card email preview (white card with #e0e0e0 border on a warm dark-mode surface) is the biggest single compositional failure — it looks like a modal inserted from a different product. The blue #60A5FA website badge is a third accent color that appears from nowhere. None of these are fatal; all are systematic enough to be fixable with one pass.

**Deterministic scan**: Detector found 1 issue at exit code 2 — `transition: width` on `.progress-fill` (globals.css:224) causes layout thrash; should use `transform: scaleX()` instead. Manual scan found the more significant issues:

- **4 files using old teal accent (#00E5CC)**: GridOverlay.tsx, DrawControls.tsx, LocationBreakdown.tsx, and globals.css line 248 (`.pill--enriching` background). These are holdovers from the previous design system; the whole UI except the map layer was migrated to amber.
- **1 glassmorphism violation**: `.glass` class (globals.css:118) has a cold blue-gray background `rgba(22,25,32,0.72)` — not warm, breaks the palette temperature rule.
- **1 side-stripe violation**: LeadQueue.tsx:254, `borderLeft: 3px solid var(--accent)` as active-row indicator.
- **7 hardcoded hex values** in components (AreaStats, ExportModal, LeadQueue, BusinessContext, EmailComposer preview).
- **23 uppercase label instances** — systematic across every panel.
- **0 ghost-cards** via automated detection (the email preview's white card is in TSX inline styles, below the detector's CSS-only scan).

No false positives identified. The detector correctly scoped to CSS and missed the inline TSX violations, which the manual scan covered.

## Overall Impression

The Darkroom concept is coherent and distinctive — warm near-black surfaces, a single amber safelight, JetBrains Mono for data, draft provenance badges. That foundation is real and worth protecting. The failure is that two separate design system migrations were left half-finished: the map layer still runs on the old teal system, and the uppercase-everywhere scaffold from the initial build was never cleaned up. Fix those two things and the interface jumps from "AI template feel" to something that genuinely looks considered.

The single biggest opportunity: the map grid overlay. It's the first thing you see when the tool opens — a teal grid on a dark map. That teal color directly contradicts the entire warmth-through-restraint brand. Changing four files to use `var(--accent)` would make the product feel unified in a way no other single change could.

## What's Working

**1. Draft provenance and save-state badges.** "AI draft · saved", "Draft saved", "AI draft · edit freely" — these clearly communicate what the email is and whether it's persisted. That's a real design decision, not a default.

**2. Keyboard shortcuts (S/X/R) for lead workflow.** The power-user path exists and works. The `tag !== 'INPUT'` guard prevents conflicts with search inputs.

**3. Lead queue status icons.** The ✏ / 🟢 / ⚠ glanceable indicators and the amber left-border active state convey information without text clutter.

## Priority Issues

**[P0] Teal accent (#00E5CC) still active on map layer**
- **What**: GridOverlay.tsx, DrawControls.tsx, LocationBreakdown.tsx, and globals.css:248 all use the old teal accent. When the map is open, the grid glows cyan — directly contradicting every other amber interactive element on screen.
- **Why it matters**: The map is the first view the user sees. Two competing saturated colors violate the One Safelight Rule and destroy brand coherence. A user's eye lands on teal, then scans right to amber — nothing resolves.
- **Fix**: Replace all `#00E5CC` and `rgba(0,229,204,...)` with `var(--accent)` and `var(--accent-dim)`. In GridOverlay.tsx and DrawControls.tsx, change cell/polygon stroke to `#E8930A`. Update globals.css `.pill--enriching` background from `rgba(0,229,204,0.1)` to `var(--accent-dim)`. Also update `.claude/rules/ui.md` — it still documents the old teal system, which will cause future regressions.
- **Suggested command**: `$impeccable polish client/src/components/Map`

**[P0] Uppercase eyebrow labels on every section (23 instances)**
- **What**: Every panel, every section header, every status group has `textTransform: uppercase` + 11px + letterSpacing 0.08em as scaffolding. Files: EmailComposer, FilterPanel, LeadQueue, ResultsTable, ResultsPanel, LocationBreakdown, BusinessContext, AreaStats, JobProgress, BusinessTable, ExportModal, SearchPanel, Sidebar.
- **Why it matters**: This is the most visible AI-template tell in the system. It signals "I couldn't figure out hierarchy so I made everything a label." With 23 instances across 12 files, it's pervasive.
- **Fix**: Keep uppercase for `.pill` text (correct, deliberate) and `.bento-label` (correct — 11px stat cell labels, not section headers). Remove `textTransform: uppercase` from all section/panel/group headers. Use weight contrast (font-weight 500 vs 400) and color contrast (var(--text-muted) vs var(--text-secondary)) to establish hierarchy instead. The `.bento-label` and `.pill` patterns in globals.css are fine as-is; it's the ad-hoc inline copies in components that need stripping.
- **Suggested command**: `$impeccable typeset`

**[P0] .glass class uses cold blue-gray background**
- **What**: globals.css:118, `.glass { background: rgba(22, 25, 32, 0.72) }`. RGB (22,25,32) is a cold blue-tinted near-black — the opposite of the warm amber-tinted surfaces everywhere else.
- **Why it matters**: Any element using the `.glass` class breaks the warm canvas rule. Even if `.glass` is used sparingly, having it wrong in the system is a latent correctness bug.
- **Fix**: Change to `rgba(26, 22, 16, 0.72)` — which is `--bg-panel` (#1A1610) at 72% opacity. This stays warm.
- **Suggested command**: `$impeccable polish client/src/styles/globals.css`

**[P1] No confirmation before sending email**
- **What**: Outreach.tsx fires `onSend()` directly on button click. A mis-click sends a real cold email to a real business.
- **Why it matters**: This is the highest-stakes action in the tool. The cost of an accidental send (damaged reputation, wasted quota) is much higher than the cost of one extra click. The daily cap means mistakes compound.
- **Fix**: Add an inline confirmation state in EmailComposer: on first Send click, expand the button area to show "Send to [email]? [Confirm] [Cancel]" inline (no modal needed). Second click sends. Alternatively, require holding the S key for 500ms. Either approach eliminates mis-clicks without interrupting the workflow.
- **Suggested command**: `$impeccable harden client/src/components/Outreach`

**[P1] BusinessContext website badge uses #60A5FA (Vercel sky blue)**
- **What**: BusinessContext.tsx:191, `color: '#60A5FA'` — a cool blue that's not in the design system and directly references the Vercel/Railway anti-aesthetic.
- **Why it matters**: Third accent color violation. Breaks the One Safelight Rule.
- **Fix**: Change to `background: 'var(--accent-dim)', color: 'var(--accent)'` for consistency, or use `var(--text-secondary)` if this shouldn't be an accent-level indicator.
- **Suggested command**: `$impeccable polish client/src/components/Outreach/BusinessContext.tsx`

**[P1] Ghost-card email preview (white card on dark surface)**
- **What**: EmailComposer.tsx:154-173 renders the email preview with `background: '#ffffff'`, `border: '1px solid #e0e0e0'`, `color: '#1A1A1A'`. White card floating on dark surface.
- **Why it matters**: Breaks dark-surface rule. Looks like a modal from a different product was dropped into the Darkroom UI. Also uses Arial/system fonts inconsistently.
- **Fix**: The preview intent (simulating how email renders in inbox) is valid — keep the light appearance but contain it properly. Wrap in a narrow inset frame with a label "Email preview" above, and use a slightly warm white (`#F5F0E8` matching warm paper) instead of pure `#ffffff`. Keep text in Outfit, not Arial. This preserves the visual metaphor without the compositional shock.
- **Suggested command**: `$impeccable polish client/src/components/Outreach/EmailComposer.tsx`

## Persona Red Flags

**Alex (Power User)**
- No batch lead actions: can't select 5 leads and mark all "contacted" at once. One-item-at-a-time.
- Named filter sets don't exist: filters reset on reload. Can't save "high-rating restaurants, no website" for daily reuse.
- Shortcut legend buried in 11px fine print — no cheat sheet, no `?` hotkey, no button hints like "Send [S]".
- No column reordering in BusinessTable.

**Sam (Accessibility-Dependent)**
- Emoji status indicators (✏ ⚠ 🟢) in LeadQueue:343-356 have no ARIA labels. Screen reader announces "pencil emoji", not "has draft".
- Email validity conveyed by dot color only (green/orange/red). Color-blind users get no fallback text.
- ResultsPanel and LeadQueue search/select inputs have `outline: none` with no replacement focus style. Tab navigation produces invisible focus.
- Nested location selectors (▾ ▸) are plain divs with click handlers — not `<button>` elements, no `aria-expanded`.
- Silver Halide (#9A8F7E) on bg-panel (#1A1610) = 4.2:1 — passes AA, but at the floor. Any slightly lighter application fails.

**Riley (Stress Tester)**
- Clearing filters while on page 3-of-5 doesn't reset pagination. User sees page 3-of-2 — invalid state.
- Switching leads in Outreach.tsx:201 clears draft without confirmation. Switching accidentally = losing work.
- Network error during send: EmailComposer removes lead from queue after 800ms timeout regardless of send success. Failed sends lose context silently.
- No loading state in LeadQueue while fetching — component shows empty state before data arrives; user can't distinguish "empty" from "loading".

## Minor Observations

- `transition: width` on `.progress-fill` (globals.css:224) — layout thrash. Switch to `transform: scaleX()` with `transform-origin: left`.
- `dangerouslySetInnerHTML` in EmailComposer:179 for signature — XSS risk if signature source is not sanitized upstream.
- Draft auto-save debounce is 1500ms (Outreach.tsx:192) — visible lag. 800ms would feel more responsive.
- Copy-to-clipboard in BusinessTable:124-128 silently swallows failures (`.catch(() => {})`). Should show "Copy failed" for 1 second.
- `useRef` timeout cleanup on unmount missing in EmailComposer send-pulse — minor leak.
- Pagination display uses hyphen "1-25 of 500" — use en-dash "1–25 of 500".
- `.claude/rules/ui.md` documents the OLD teal system (`--accent: #00E5CC`) — will cause future regressions if any AI session reads it without also reading DESIGN.md. Needs updating.
